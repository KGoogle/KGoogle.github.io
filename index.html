암호 변환기

[인코딩 알고리즘]
각 음절 ('가'~'힣')은 유니코드 순서에 따라 0~11171의 고유한 인덱스를 갖습니다.
( 인덱스=(초성 번호×21×28)+(중성 번호×28)+종성 번호 )
정보 손실을 막기 위해 인덱스에 1을 더한 1~11172 범위의 값을 사용합니다.
( vi = 인덱스+1 )
이 값을 0이 없는 11172진법의 자릿값으로 하여 문자열을 하나의 10진수로 변환합니다.
( 숫자 = v₁ × 11172ⁿ⁻¹ + v₂ × 11172ⁿ⁻² + ⋯ + vₙ × 11172⁰ = ∑i=1n vi×11172n−i )
[디코딩 알고리즘]
인코딩된 숫자 N으로부터 복원된 음절들은 추출 순서의 역순으로 정렬하여 문자열을 만듭니다.
인코딩 시 더해진 1을 보정하기 위해, N에서 1을 뺀 값을 11172로 나눕니다.
이때의 나머지는 '인덱스'가 되고, 몫으로 N값을 갱신하며 이 과정을 반복합니다.
   반복: N > 0인 동안
   보정 값 = N - 1
   인덱스 = 보정 값 mod 11172
   음절 = chr(0xAC00 + 인덱스)
   N = 보정 값 ÷ 11172 (정수 나눗셈)
